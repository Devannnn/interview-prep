# How do you validate data received by an API?

## Why recruiters ask this
- Assesses data quality discipline and defensive coding.
- Critical for long-term reliability of shared APIs.

## Answer structure (keep it concise)
1. Validate schema, types, required fields, and constraints.
2. Compare approaches: manual validation vs Marshmallow/Pydantic.
3. Return clear, consistent validation errors (`400`/`422`).
4. Mention how validation protects downstream services and storage.
