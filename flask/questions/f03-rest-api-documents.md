# How do you structure a REST API for a document system?

## Why recruiters ask this
- Evaluates API design clarity and consistency.
- Highly relevant for repository/document platforms.

## Answer structure (keep it concise)
1. Define core resources and endpoints (`/documents`, `/documents/{id}`).
2. Map CRUD operations to HTTP methods consistently.
3. Explain status codes, validation, and error formats.
4. Mention versioning, pagination, and filtering from the start.
